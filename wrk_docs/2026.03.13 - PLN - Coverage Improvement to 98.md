# Coverage Improvement Plan to 98

Date: 2026-03-13
Project: `mdmdview`

## Goal

Improve coverage in measured stages, with branch coverage treated as the gating metric.

Two end states are possible:

1. Main application source reaches 98%+ across lines, functions, regions, and branches.
2. Whole workspace reaches 98%+ across the same metrics.

These are not the same amount of work. The baseline gap made that explicit, and the stage 2 measurement correction made it even clearer.

Current interpretation:

- whole-workspace 98% remains a major project because `crates/d2` and `src/bin/full_test.rs` still dominate uncovered production branches
- `src/` to 98% is much more tractable and is now the sensible near-term execution target unless Arthur explicitly wants repo-wide closure

## Execution Rules

For each stage:

1. Add the smallest high-yield tests first.
2. Run:
   - `cargo fmt --all`
   - `cargo clippy --workspace --all-targets -- -D warnings`
   - `cargo test -q`
   - `cargo +nightly llvm-cov --workspace --all-features --branch ...`
3. Save updated coverage artifacts and journal notes.
4. Commit the stage atomically.

- `src/bin/full_test.rs` alone still carries 119 uncovered branches
- `crates/d2/src/compiler.rs`, `edge_routing.rs`, `svg_render.rs`, `parser.rs`, and `shapes.rs` account for most of the remaining workspace branch debt
- the remaining gap to whole-workspace 98% is no longer a cleanup pass; it is a dedicated testing project across the embedded `d2` engine and the `full_test` utility binary

Recommended continuation from here:

1. Decide whether coverage policy should be enforced on:
   - whole workspace, or
   - production application sources only
2. If the answer is whole workspace, split stage 5 into:
   - `full_test.rs`: decide whether to test it heavily or exclude it from policy
   - `crates/d2` parser / compiler negative-path campaign
   - `crates/d2` layout / svg / shapes visual-edge campaign
3. If the answer is application sources only, the 98% branch target is already met and future work should focus on keeping it from regressing
