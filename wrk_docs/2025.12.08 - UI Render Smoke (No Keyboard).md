# UI Render Smoke (No Keyboard)

**Date:** 2025-12-08  
**Scope:** Verify visual rendering (fonts, tables, images, code) after table width persistence fix, without using keyboard interactions.  
**Build:** target/release/mdmdview.exe  
**Tools:** mdscreensnap.exe for captures.

## Test Matrix

| ID | File | Purpose | Pass Criteria |
| --- | --- | --- | --- |
| T1 | examples/table_test.md | Table layouts & column widths | Headers/body aligned; no tofu; widths proportional; check persistence visually after reopen. |
| T2 | examples/complete_test.md | Mixed markdown coverage | Headings, lists, code, links render correctly; no clipping/tofu. |
| T3 | examples/inline_images.md | Image rendering | Images visible, no missing glyphs or broken icons. |
| T4 | examples/syntax_test.md | Syntax highlighting | Code blocks use expected colors; text legible. |
| T5 | examples/list_test.md | Lists & bullets | Indentation and markers render cleanly; spacing consistent. |

## Procedure

1) Ensure release build exists: `cargo build --release` (already built earlier).  
2) For each test ID:  
   - Launch `mdmdview.exe <file>`  
   - Wait ~4 seconds for render.  
   - Capture screenshot with `mdscreensnap` named after the test ID.  
   - Close mdmdview process.  
3) Inspect screenshots for:  
   - Font rendering issues (tofu, jagged edges).  
   - Correct layout/spacing.  
   - Artifacts or truncation.  
4) Record observations below.
