## Executive Summary

The repository is still nowhere near a literal 98% whole-workspace coverage target.

Important interpretation detail:

- Stage 1 added real renderer UI-path tests and raised both `src/` and workspace coverage.

- That improved branch fidelity, but it also reduced line/function percentages in scopes that had previously been inflated by test code being counted as covered production code.

The near-term path to 98 is realistic only for the main application source tree, not for the whole workspace.

## Blocker Found and Fixed

The initial clean test run exposed one deterministic failure:

- `app::tests::test_save_current_document_parentless_path`

Root cause:

- The test wrote a relative file directly into the repo root.
- In this sandbox, that path can return `Access is denied` even when Cargo can build successfully in the alternate writable root.

Fix:

- Added a test-only current-directory guard in [C:\language\mdmdview\src\app.rs](C:\language\mdmdview\src\app.rs)
- Updated the test to run the parentless-path scenario inside a temporary working directory while preserving the same branch behavior

## Concrete Findings

1. Stage 1 worked. `src/d2_renderer.rs` and `src/pikchr_renderer.rs` are no longer broad missing-test problems.
2. The remaining misses in both renderer wrappers are concentrated in:
   - `poll_results`
   - `begin_frame`
   - `render_block`
   - debounce / stale-texture branches when stale textures are absent or invalid
   - invalid texture-key staleness fallbacks
3. `src/markdown_renderer.rs`, `src/app.rs`, and `src/mermaid_renderer.rs` are branch-gap problems, not broad missing-test problems.
4. Stage 2 corrected a real instrumentation issue: some prior percentages were overstated because inline test modules were still counted as covered code.
5. Whole-workspace 98% is not a cleanup pass. It is a dedicated campaign across `crates/d2` and the `src/bin/full_test.rs` utility binary.

## Recommendation

Implementation should proceed in this order:

1. Finish the remaining source-path gaps in `src/d2_renderer.rs` and `src/pikchr_renderer.rs`.
2. Close the remaining branch-heavy gaps in `src/markdown_renderer.rs`, `src/app.rs`, and `src/mermaid_renderer.rs`.
3. Decide explicitly whether the target scope is:
   - whole workspace, or
   - main application source

Without that scope decision, a statement like "continue until coverage reaches 98%" is ambiguous enough to waste a lot of effort.
