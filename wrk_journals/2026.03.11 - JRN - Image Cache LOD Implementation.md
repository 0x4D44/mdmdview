# Journal - Image Cache LOD Implementation

## 2026-03-11

### Plan

Implement the LOD (Level-of-Detail) image cache redesign per the HLD at
`wrk_docs/2026.03.11 - HLD - Image Cache LOD.md`.

**Stages:**

1. **Stage 1 — ImageCacheEntry + ImageCache structural changes**
   - Add `degraded: bool` to `ImageCacheEntry`
   - Add `degrade()` method to `ImageCache`
   - Add `enforce_budget()` method to `ImageCache`
   - Remove eviction loop from `insert()` (keep entry-count safety valve)
   - Remove `pick_eviction_candidate()`
   - Remove `touch()` from `get()` and delete `touch()`
   - Write tests for degrade, enforce_budget, insert without eviction
   - **Commit**

2. **Stage 2 — Rendering path: degraded reload logic**
   - Modify `get_or_load_image_texture` to detect degraded entries
   - Add budget check before enqueuing reload of degraded entry
   - Use previous-frame viewport distances for reload-zone decision
   - Handle degraded+stale entries (don't remove, enqueue reload)
   - Write tests
   - **Commit**

3. **Stage 3 — Frame lifecycle: call enforce_budget**
   - Call `enforce_budget()` at end of `render_to_ui()` after viewport
     distances are computed
   - Pass `egui::Context` through to the cache
   - Wire up the full frame sequence: poll → render → collect → distances → enforce
   - Write integration tests
   - **Commit**

4. **Stage 4 — Cleanup + final review**
   - Remove dead code (pick_eviction_candidate, old eviction tests)
   - Update any status bar / tooltip code if needed
   - Full build + test pass
   - Code review + devils advocate
   - **Commit**
