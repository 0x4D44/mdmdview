## Review Findings — Consolidated

### HIGH (fixed)

**H1. User drag-to-resize doesn't stick** → Fixed: natural_widths made purely content-driven (removed max(content, policy_width)). Proportional allocation now locks persisted columns and distributes remaining space among non-persisted columns.

**H2. Proportional allocation overflow** → Fixed: defensive scaling block after allocation — if total > available, scale all columns down proportionally.

### MEDIUM (fixed)

**M1. Proportional runs when content_fits** → Fixed: guard includes `&& !content_fits`.

**M2. Dead parameters** → Fixed: removed `_scaled_down` from build_table_column_layout, `_min_floor` from resolve_table_column_widths, 4 dead params from estimate_table_row_widths, `tooltip` from ColumnSpec, `min_floor`/`available_for_columns`/`scaled_down` from TableLayoutPlan.

**M3. all_indices Vec** → Fixed: replaced with direct iteration `0..column_specs.len()` and `min_widths.iter().sum()`.

**M4. Code quality** → Fixed: renamed `fixed_spec` → `spec_with_min`, removed `_preferred` from `resizable_spec`, fixed unused variable warning.

### Not fixed (acceptable)

- Hash stability across upgrade — one-time reset, acceptable in dev sandbox
- ColumnPolicy single-variant enum could be struct — cosmetic, deferred
- px() only used in one test — marginal, kept for coverage of normalize_body_font_px

## Summary of all changes

### `src/table_support/column_spec.rs`
- **ColumnPolicy**: collapsed from 4 variants to `Resizable { min: f32 }`
- **Deleted**: classify_column, column_needs_remainder, matches_any, column_tooltip, MAX_REMAINDER_COLUMNS, apply_preferred_width
- **ColumnSpec**: removed `tooltip` field and parameter from `new()`
- **derive_column_specs**: ~90 lines → ~25 lines. Simple loop computing min from header width. No keyword matching, no remainder tracking.
- **Tests**: 25 deleted, 6 added, 12 kept = 21 total (all pass)

### `src/markdown_renderer.rs`
- **build_table_column_layout**: all three branches unified — Column::initial(width).at_least(min).resizable(true), no clip. Removed `scaled_down` param.
- **compute_table_layout_plan**: removed fixed/flexible/remainder separation. Proportional allocation for ALL columns. Persisted widths respected (locked columns, remaining space distributed). Defensive overflow scaling. Guard excludes content_fits. Removed apply_persisted_widths call.
- **resolve_table_column_widths**: unified logic — persisted > stored > min. Removed `min_floor` param.
- **estimate_natural_column_widths**: purely content-driven — removed `resolved_widths` param and max(content, policy).
- **estimate_table_row_widths**: simplified from 8 params to 3.
- **persist_resizable_widths**: removed Resizable-only guard.
- **apply_persisted_widths**: deleted entirely.
- **TableLayoutPlan**: removed `min_floor`, `available_for_columns`, `scaled_down` fields.
- **Test helpers**: renamed fixed_spec → spec_with_min, removed _preferred from resizable_spec.
- **Tests**: ~10 rewritten, all 475 pass.
