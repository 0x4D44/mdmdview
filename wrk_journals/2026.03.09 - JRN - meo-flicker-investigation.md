- Hard findings:
  - `ENQUEUED` proves the disappearing heart image is a cache miss path, not the timestamp-staleness path.
  - In this codebase, a previously displayed image can only become a cache miss if it was removed from `image_textures` elsewhere: either `release_gpu_textures()` or `ImageCache::insert()` eviction.
  - The screenshot cache line (`14 entries`, `99.0/100.0 MB`) strongly points at eviction pressure rather than a full texture release.
  - The cache entry count is higher than the sample document should produce by itself, which points to cross-document/path cache pollution or duplicate keys in addition to raw memory pressure.
- Code changes made:
  - Added an image-load generation counter so stale background results are ignored after a document change.
  - Added `reset_image_state()` and wired `load_content()` to clear per-document image runtime state on new content loads.
  - Added tests covering stale-result rejection and runtime image-state reset.
- Emotional temperature:
  - Relief. The screenshots finally cut through the earlier false trail.
  - Still cautious: if a single document genuinely needs more than the cache budget, viewport-aware image loading is still the real long-term fix.
